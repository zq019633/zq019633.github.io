[_tableView reloadData];

2.如果要说使用didSelectRowAtIndexPath方法 需要指向那个代理
_tableView.dataSource = self;
_tableView.delegate = self;

NSMutableArray as=[[NSMutableArray alloc] init];

ios中不同的controller传递数据---代理方式

ios开发中会出现2个controller 传递数据的情况，和安卓中的两个activity传递数据是一个道理，只不过安卓中是通过Intent传递，ios中我使用的是代理的方式传递数据。
一般都是AViewController 把数据传递到NextViewController

第一步：
在A.h中：
@class AViewController;
@protocol AViewCOntrollerDelegate

//这个方法是用来在BViewController中接收数据
-(void) passData:(NSDictionary *)values;

@end

并且在A.h中声明一个代理
@property (nonatomic, weak) id delegate;

第二步：
在A.m中 要跳转的地方（一般都是UITableView的条目 响应点击事件）

-(void)tableView:(UITableView )tableView didSelectRowAtIndexPath:(NSIndexPath )indexPath
{

NSLog(@"我要跳转了");

NextViewController *next1=[[NextViewController alloc] init];
self.delegate=next1;

//这里的array是通过网络请求获得的
[self.delegate passData:[array objectAtIndex:indexPath.row ]];
[self.navigationController pushViewController:next1 animated:YES];

}

第三步：

在NextViewController.h 中 实现这个代理

@interface NextViewController : UIViewController{
}

然后在在NextViewController.m中实现passData方法，获取从AViewController中传递过来的数据

-(void) passData:(NSDictionary *)values{

NSLog(@"我被传过来了%@",values);

}
⚠️ 传递的时候要以字典的形式 也就是NSDictionary 比较方便
